# PaperRev 공동 작업자 공유용 브리프

업데이트: 2026-05-30 KST

## 한 줄 목표

PaperRev는 Gerontology/Aging 분야의 최신 논문을 자동 수집한 뒤, 노화심리 하위 분야별 카드뉴스와 이론 해설로 정리해 보여주는 공개용 큐레이션 페이지입니다.

## 왜 만드는가

최신 논문은 계속 나오지만, 독자가 원문 목록을 직접 뒤져 흐름을 파악하기는 어렵습니다. PaperRev의 목표는 학술 검색엔진을 하나 더 만드는 것이 아니라, 사용자가 "이번 주 노화심리 연구에서 무엇을 먼저 보면 되는지" 빠르게 이해할 수 있는 읽기 경험을 제공하는 것입니다.

## 현재 구현 상태

- 정적 사이트로 구현되어 GitHub Pages 배포에 적합합니다.
- 메인 화면은 분야별 카드뉴스, 핵심 질문, 노화 이론 해설을 중심으로 구성됩니다.
- 논문은 7개 분야로 묶습니다: 인지노화, 정서노화, 사회적 노화, 치매·인지장애, 돌봄·가족, 건강행동·디지털헬스, 일·은퇴·연령주의.
- 메인에는 자동 발행 기준을 통과한 분야별 대표 카드뉴스 5개씩, 총 35개를 노출합니다.
- 메인에 포함되지 않은 수집 논문은 `archive.html` 보관함에서 검색/필터로 확인합니다.
- 노화 이론 섹션은 최신 논문을 해석하기 위한 배경 층으로, 이론별 핵심 주장, 대표 개념, 한계, 관련 이론, anchor 문헌을 제공합니다.
- 각 이론 상세에는 현재 featured 카드뉴스 중 관련도가 높은 논문 3개를 자동 연결해 보여줍니다.

## 현재 데이터 스냅샷

2026-05-30 현재 파일 기준:

- 탐색 창: 최근 30일
- 총 수집 논문: 340편
- 자동 발행 후보: 176편
- 리뷰 보류: 164편
- 메인 카드뉴스: 35개
- 보관함 논문: 305개
- 보류 사유: 초록 없음, 낮은 featured 점수, 낮은 분류 신뢰도, 분야 중복 등

## 데이터 흐름

1. `scripts/build-drafts.mjs`가 allowlist 저널의 최신 논문을 Crossref에서 수집합니다.
2. DOI 기준으로 중복을 제거하고, 제목/초록 키워드로 분야와 독자 질문을 자동 분류합니다.
3. OpenAlex와 PubMed 링크를 보강해 원문/메타데이터 접근 경로를 넓힙니다.
4. `publishStatus`, `needsReview`, `qualityFlags`를 기준으로 자동 발행 후보와 리뷰 보류 항목을 분리합니다.
5. `data/drafts/featured-drafts.json`은 메인 카드뉴스 전용 데이터로 사용합니다.
6. `data/drafts/archive-records.json`은 보관함 전용 데이터로 사용합니다.
7. `data/drafts/article-drafts.json`은 전체 묶음과 자동화 호환용으로 유지합니다.
8. `data/drafts/featured-curation.json`은 자동 발행을 막는 검수 파일이 아니라, 해당 ID가 있을 때 한국어 문구를 더 자연스럽게 덮어쓰는 선택적 보정 파일입니다.

## 운영 원칙

- 자동 발행을 기본값으로 두고, 문제가 감지된 항목만 `reviewRecords`로 보류합니다.
- `reviewRecords`는 공개 메인 화면에 섞지 않습니다.
- 영어 초록 문장을 그대로 화면에 노출하지 않고, 한국어 카드뉴스 문장으로 정리합니다.
- 카드뉴스는 원문을 대체하는 최종 해석이 아니라, 어떤 논문을 먼저 볼지 돕는 안내층입니다.
- 임상적 조언처럼 읽히는 문장은 피하고, 원문/DOI/Crossref/OpenAlex/PubMed 링크를 함께 제공합니다.
- 보관함은 심화 탐색용 보조 뷰로 두고, 메인 경험은 간결하게 유지합니다.

## 최근 개선 사항

- 자동 발행/리뷰 보류 정책을 추가했습니다.
- 메인 카드뉴스는 `auto_publish` 항목만 사용하도록 제한했습니다.
- `missing_abstract`, `low_classification_confidence`, `topic_overlap`, `low_featured_score` 등 품질 플래그를 저장합니다.
- 보관함은 한 번에 전체를 렌더링하지 않고, 기본 30개 표시 후 `더 보기`로 확장합니다.
- 메인과 보관함 데이터 파일을 분리해 초기 로딩 부담을 줄였습니다.
- 분야별 보조 태그(`secondaryTopicIds`)를 표시해 교차 주제 논문을 더 자연스럽게 설명합니다.
- 이론별 관련 분야, 독자 질문, 키워드, OpenAlex 개념을 점수화해 최신 카드뉴스와 자동 연결합니다.
- 코드 문법 검증과 로컬 HTTP 응답 확인을 마쳤습니다.

## 공동 작업자에게 부탁할 검토

1. 메인 카드뉴스 35개가 노화심리 독자가 먼저 볼 만한 항목인지 봐 주세요.
2. 7개 분야 구분이 직관적인지, 분야명이 외부 독자에게 어색하지 않은지 의견을 주세요.
3. 카드뉴스 문장이 너무 단정적이거나 원문 확인 없이 결론처럼 읽히는 부분이 있는지 확인해 주세요.
4. 보관함이 "더 찾아보고 싶은 사람"에게 충분한 보조 탐색 도구인지 봐 주세요.
5. 노화 이론 섹션이 최신 논문을 읽는 데 실제로 도움이 되는지, 더 필요한 이론이나 anchor 문헌이 있는지 제안해 주세요.
6. 이론 상세에 자동 연결된 카드뉴스 3개가 해당 이론으로 읽기에 자연스러운지 확인해 주세요.

## 바로 다음 작업

우선순위는 협업자 검토를 통해 `이론-논문 자동 연결`의 어색한 매칭을 보정하는 것입니다. 현재 연결은 규칙 기반이므로, 특정 이론에 너무 넓거나 좁게 연결되는 카드가 있으면 `THEORY_ARTICLE_RULES`의 키워드를 조정하면 됩니다.

그 다음 단계는 OpenAI API 기반 카드뉴스 문장 개선입니다. 이때도 `auto_publish` 후보에만 비용을 쓰고, `needs_review` 항목은 보류 큐에 남기는 방향이 적절합니다.

## 공동 작업자가 보면 좋은 파일

- `index.html`, `app.js`, `styles.css`: 메인 페이지 구조와 렌더링
- `archive.html`, `archive.js`: 수집 보관함 뷰
- `scripts/build-drafts.mjs`: 수집, 분류, 품질 플래그, draft 생성 로직
- `data/drafts/featured-drafts.json`: 메인 카드뉴스 전용 데이터
- `data/drafts/archive-records.json`: 보관함 전용 데이터
- `data/drafts/article-drafts.json`: 전체 draft 묶음과 리뷰 보류 기록
- `data/drafts/featured-curation.json`: 메인 카드뉴스 한국어 문구 보정
- `data/theory-notes.json`, `data/theory-papers.json`: 노화 이론 설명과 anchor 문헌
